不吐不快——吐槽Tomcat与Eclipse相关联过程中遇到的坑

磕磕绊绊花了一两个小时总算在eclipse里把Tomcat配置好了

墙裂吐槽一下其中很多时间花费在等待下载安装eclipse及插件上了，动不动就40、50kb/s是什么鬼啊

​

首先要选择菜单栏Windows–>preferences，找到其中的Server项

结果却发现我的eclipse没有server这一栏？？？

​

查了半天，发现是要装一个插件，好吧那就装吧

装完之后，好了，server有了

进入之后选择Tomcat的版本，最高版本只有7.0？？

可我明明装的是9.0啊？？？

又查查查，试了很多种方法，安装了插件发现也没有任何作用，可供选择的版本最高还是7.0

​

过了好一会儿，才知道好像是eclipse的版本装错了……

在安装eclipse时有很多版本可供选择，因为 一开始在学Java的基础语法，所以当时我选择的是第一个最普通的JavaSE版本

而进行JSP开发需要选择第二个JavaEE版的

​

好吧，删除了现有版本又重新开始安装

安装完成后，打开eclipse，选择一个workspace

因为比较懒，所以没有新建一个文件夹，而是选择了原版本eclipse创建的一个workspace

​

进去之后，一切顺利，server也有了，Tomcat最高版本也是9.0

正当我欢欣鼓舞之时，却突然发现……

在File–>New里找不到Dynamic Web Project？？？

什么鬼？？？环境都配置好了，连创建新项目的机会都不给？？？

瞬间崩溃……

​

正当我百思不得其解抓耳挠腮之时，突然灵光一闪

想到是不是跟最初选择的workspace有关呢？

因为我们可以清楚的看到，新建了一个workspace之后，即使没有进行任何操作

那个文件夹里也会出现名为 .metadata 、 RemoteSystemsTempFiles 的默认初始化文件夹

是不是这些文件夹影响到了eclipse的配置呢？

​

于是重新打开eclipse，创建了一个新的文件夹作为workspace

点开File–>New

终于！！！出现了Dynamic Web Project 选项！！！

啊……总算松了一口气

整个人葛优瘫在椅子上……

​

细细回想，也多亏了当时突然迸发出的想法，才能摆脱这种连续不断的困境

这种完全依靠以往的经验以及各种胡思乱想而在一瞬间得出的想法，真是太奇妙了……
